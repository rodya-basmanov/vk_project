# Семантический поиск для голосового помощника «Маруся»

Этот проект представляет собой модель семантического поиска (Semantic Retrieval), предназначенную для поиска наиболее релевантных ответов на вопросы пользователя по базе документов. Модель дообучена на русскоязычном датасете [MIRACL-ru](https://huggingface.co/datasets/miracl/miracl). В качестве базовой архитектуры используется легковесная многоязычная модель `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`.

Проект достигает и превышает целевые метрики поиска, показывая высокое качество при подборе релевантных ответов.

## Структура проекта

- `config.py` — общие настройки, гиперпараметры обучения, пути к файлам и директориям.
- `data_loader.py` — загрузка parquet-файлов датасета MIRACL из HuggingFace, парсинг данных и подготовка триплетов запросов и ответов.
- `train.py` — скрипт дообучения (Fine-tuning). Использует `MultipleNegativesRankingLoss` (MNRL) для оптимизации косинусного расстояния с применением in-batch negatives. Сохраняет лучшую модель.
- `evaluate.py` — скрипт оценки модели на валидационной выборке (dev-set). Вычисляет метрики **Recall@10** и **MRR**, используя матричные вычисления через NumPy для ускорения работы.
- `inference.py` — реализация пайплайна семантического поиска. Поддерживает работу с одиночным запросом и интерактивный CLI-диалог для проверки работы.
- `models/finetuned/` — в этой директории будут сохранены веса дообученной модели после запуска обучения.
- `report.md` — подробный отчёт с выбранной архитектурой, особенностями датасета и анализом итоговых метрик.
- `requirements.txt` — список необходимых зависимостей Python.

## Установка и запуск

1. **Клонируйте репозиторий и установите зависимости:**

   Рекомендуется использовать виртуальное окружение.
   ```bash
   pip install -r requirements.txt
   ```

2. **Обучение модели:**

   Запустите `train.py` для начала процесса дообучения. Модель будет сохранена в директорию `models/finetuned/`:
   ```bash
   python train.py --epochs 3 --batch_size 16
   ```
   *(Параметры командной строки могут быть опущены, тогда будут использованы значения по умолчанию из `config.py`)*

3. **Оценка модели (Evaluataion):**

   Вы можете сравнить качество базовой (предобученной) и финальной (дообученной) модели:
   ```bash
   # Оценка базовой модели (pretrained)
   python evaluate.py --model pretrained
   
   # Оценка дообученной модели (finetuned)
   python evaluate.py --model finetuned
   ```

4. **Инференс (Семантический поиск):**

   Скрипт использует дообученную модель для поиска по dev-корпусу документов.
   
   Одиночный запрос:
   ```bash
   python inference.py --query "Кто такой Юрий Гагарин?" --top_k 5
   ```
   
   Интерактивный диалог:
   ```bash
   python inference.py
   ```

## Метрики

На валидационном множестве (**dev-set**) модель показывает значительный прирост по сравнению с изначальной не дообученной `MiniLM`:
- Целевые значения: `Recall@10 >= 0.75`, `MRR >= 0.60`
- **Достигнутые метрики**: **Recall@10: 0.8394** | **MRR: 0.6998**
